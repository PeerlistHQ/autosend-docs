## Request Format

The request body is the same as the send email endpoint, except `to` is an array of recipient objects:

```json
{
  "to": [
    {
      "email": "customer1@example.com",
      "name": "Jane Smith"
    },
    {
      "email": "customer2@example.com",
      "name": "John Doe"
    },
    {
      "email": "customer3@example.com",
      "name": "Bob Johnson"
    }
  ],
  "from": {
    "email": "hello@mail.yourdomain.com",
    "name": "Your Company"
  },
  "subject": "Welcome to Our Platform!",
  "html": "<h1>Welcome, {{name}}!</h1><p>Thanks for signing up.</p>",
  "dynamicData": {
    "name": "Valued Customer"
  }
}
```

## Using Templates

You can also use templates with the bulk send endpoint:

```json
{
  "to": [
    {
      "email": "customer1@example.com",
      "name": "Jane Smith"
    },
    {
      "email": "customer2@example.com",
      "name": "John Doe"
    }
  ],
  "from": {
    "email": "hello@mail.yourdomain.com",
    "name": "Your Company"
  },
  "templateId": "tmpl_abc123",
  "dynamicData": {
    "firstName": "Valued",
    "orderNumber": "ORD-12345",
    "orderTotal": "$99.99"
  }
}
```

## Dynamic Data & Personalization

Use Handlebars syntax for template variables, just like the send email endpoint:

```html
<h1>Hello {{firstName}}!</h1>
<p>Your order #{{orderNumber}} has been shipped.</p>
<p>Total: {{orderTotal}}</p>
```

Provide values in the `dynamicData` object (same data will be used for all recipients):

```json
{
  "to": [
    {
      "email": "customer1@example.com",
      "name": "Jane Smith"
    },
    {
      "email": "customer2@example.com",
      "name": "John Doe"
    }
  ],
  "from": {
    "email": "hello@mail.yourdomain.com",
    "name": "Your Company"
  },
  "html": "<h1>Hello {{firstName}}!</h1>",
  "dynamicData": {
    "firstName": "Jane",
    "orderNumber": "12345",
    "orderTotal": "$99.99"
  }
}
```

## Response Format

The response includes results for each recipient:

```json
{
  "success": true,
  "data": {
    "results": [
      {
        "success": true,
        "email": "customer1@example.com",
        "emailId": "email_xyz789",
        "status": "queued",
        "queuedAt": "2025-09-30T10:30:00.000Z"
      },
      {
        "success": true,
        "email": "customer2@example.com",
        "emailId": "email_xyz790",
        "status": "queued",
        "queuedAt": "2025-09-30T10:30:00.000Z"
      },
      {
        "success": false,
        "email": "customer3@example.com",
        "error": "Invalid email format"
      }
    ],
    "summary": {
      "total": 3,
      "successful": 2,
      "failed": 1
    }
  }
}
```

## Limits

- Maximum 100 recipients per request
- All other fields (`from`, `subject`, `html`, `text`, `templateId`, `dynamicData`, `replyTo`, `unsubscribeGroupId`) work exactly the same as the send email endpoint

## Error Handling

If some recipients fail validation, the successful emails will still be queued, and the failed ones will be returned with error messages. The response will include a `success` field for each recipient indicating whether the email was queued successfully.
