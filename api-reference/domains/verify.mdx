#### Authorizations
<ParamField path="Authorization" type="string | header" required>
  Bearer authentication header of the form `Bearer <token>`, where `<token>` is your API key.
</ParamField>

#### Path Parameters
<ParamField path="domainId" type="string" required>
  The MongoDB ObjectId of the domain to verify.
</ParamField>

#### Response

<Note>
  This endpoint uses a **fire-and-forget** pattern. It returns the current domain state immediately — the actual DNS and AWS SES verification checks run in the background. To detect status changes, poll `GET /domains/{domainId}` approximately every 15 seconds until `verificationStatus` reaches `VERIFIED`.
</Note>

<ResponseField name="success" type="boolean">
  Indicates whether the request was accepted.
</ResponseField>

<ResponseField name="message" type="string">
  Always `"Domain verification status updated"` on success.
</ResponseField>

<ResponseField name="data" type="object">
  The domain's state **at the time the request was received**, before the background verification completes. DNS record flags (`ownershipVerified`, `dkimEnabled`, `mailFromEnabled`, `dmarcEnabled`) only ever move from `false` to `true` — they are never reverted.

  <Expandable title="Domain object">
    <ResponseField name="id" type="string">
      Unique identifier for the domain (MongoDB ObjectId).
    </ResponseField>
    <ResponseField name="domain" type="string">
      The domain name.
    </ResponseField>
    <ResponseField name="awsRegion" type="string">
      The AWS region where SES is configured for this domain.
    </ResponseField>
    <ResponseField name="verificationStatus" type="string">
      Current overall verification status. One of `PENDING_CONFIGURATION`, `PENDING`, or `VERIFIED`. This value reflects the state before the background job completes — poll `GET /domains/{domainId}` for updates.
    </ResponseField>
    <ResponseField name="ownershipVerified" type="boolean">
      Whether the TXT ownership record has been verified.
    </ResponseField>
    <ResponseField name="dkimEnabled" type="boolean">
      Whether DKIM CNAME records have been verified.
    </ResponseField>
    <ResponseField name="mailFromEnabled" type="boolean">
      Whether the MAIL FROM MX/TXT records have been verified.
    </ResponseField>
    <ResponseField name="dmarcEnabled" type="boolean">
      Whether a DMARC TXT record has been detected.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the domain was added.
    </ResponseField>
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of the last status update.
    </ResponseField>
  </Expandable>
</ResponseField>

#### Errors

<ResponseField name="404" type="object">
  Returned when no domain with the given `domainId` exists in the project.
</ResponseField>

<ResponseField name="429" type="object">
  Returned when the verify endpoint has been called too many times in a short period. Wait before retrying.
</ResponseField>
