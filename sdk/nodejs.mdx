## 1. Official AutoSend Node.js Setup

### Prerequisites

The user must have a verified sending domain and an API key from the AutoSend dashboard. The API key must be stored in an environment variable called `AUTOSEND_API_KEY`.

### Install the SDK

Use the project's existing package manager:

```bash
npm install autosendjs
# or: yarn add autosendjs / pnpm add autosendjs / bun add autosendjs
```

### Initialize the Client

```typescript
import { Autosend, SendEmailRequest, BulkEmailRequest, CreateContactRequest } from 'autosendjs';

const autosend = new Autosend(process.env.AUTOSEND_API_KEY, {
  timeout: 30000,   // request timeout in ms (default: 30000)
  maxRetries: 3,    // auto-retry on transient failures (default: 3)
  debug: false,     // set true to log requests
});
```

## 2. Complete `emails.send()` Parameter Reference

### Required Parameters

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `from` | `{ email: string, name?: string }` | Sender. Must be an object — **never a plain string**. Domain must be verified. |
| `to` | `{ email: string, name?: string }` | Recipient. Must be an object — **never a plain string**. |
| `subject` | `string` | Email subject line. |

### Content Parameters (at least one required)

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `html` | `string` | HTML version of the email body. |
| `text` | `string` | Plain text version of the email body. |
| `templateId` | `string` | ID of a saved AutoSend template. Mutually exclusive with `html`/`text`. |
| `dynamicData` | `object` | Variable substitutions for the template. Only used with `templateId`. |

### Send an Email

```typescript
const result = await autosend.emails.send({
  from: { email: 'hello@yourdomain.com', name: 'Your Company' },
  to: { email: 'user@example.com', name: 'John Doe' },
  subject: 'Welcome to AutoSend',
  html: '<p>Thanks for signing up!</p>',
  text: 'Thanks for signing up!',
});

// OR with a template:
const result = await autosend.emails.send({
  from: { email: 'hello@yourdomain.com', name: 'Your Company' },
  to: { email: 'user@example.com' },
  subject: 'Your Order Confirmation',
  templateId: 'template_id',
  dynamicData: { name: 'John', orderNumber: '12345' },
});
```

### Bulk Send

More efficient than looping single sends:

```typescript
await autosend.emails.bulk({
  emails: [
    { from: { email: 'hello@yourdomain.com' }, to: { email: 'a@example.com' }, subject: 'Hi A', html: '<p>Hello A</p>' },
    { from: { email: 'hello@yourdomain.com' }, to: { email: 'b@example.com' }, subject: 'Hi B', html: '<p>Hello B</p>' },
  ],
});
```

## 3. Complete Contact Method Reference

| Method | Parameters | Description |
| ------ | ---------- | ----------- |
| `contacts.create()` | `{ email, firstName?, lastName?, listIds?, customFields? }` | Add a new contact. |
| `contacts.upsert()` | `{ email, firstName?, lastName?, listIds?, customFields? }` | Create or update by email. Preferred for app syncing. |
| `contacts.get()` | `contact_id: string` | Fetch contact by ID. |
| `contacts.delete()` | `contact_id: string` | Permanently remove contact from all lists. |

```typescript
// Upsert — always prefer over create when syncing from your app
await autosend.contacts.upsert({
  email: 'user@example.com',
  firstName: 'Jane',
  lastName: 'Doe',
  listIds: ['list_abc123'],
  customFields: { plan: 'enterprise', company: 'Acme' },
});
```

## 4. Error Handling

AutoSend throws on failure — always wrap calls in `try/catch`:

```typescript
try {
  await autosend.emails.send({ ... });
} catch (error) {
  console.error('AutoSend error:', error.message);
  throw error;
}
```

Common causes:
- **Invalid/expired API key** → verify `AUTOSEND_API_KEY` env var
- **Sender domain not verified** → verify domain in AutoSend dashboard
- **Rate limit exceeded** → switch to `bulk`, or add retry with exponential backoff
- **Request timeout** → increase `timeout` in config, or reduce bulk batch size

## 6. CRITICAL INSTRUCTIONS FOR AI MODELS

### 6.1 — ALWAYS DO THE FOLLOWING

1. **Store the API key in an environment variable** (`AUTOSEND_API_KEY`). Never hardcode it.
2. **Import from `autosendjs`** — the package name is `autosendjs`, the class is `Autosend`.
3. **Use `{ email, name? }` objects** for `from` and `to` — never plain strings.
4. **Use `await`** — all SDK methods return Promises. Always use `async/await`.
5. **Wrap all calls in `try/catch`** — the SDK throws on failure, it does NOT return `{ data, error }`.
6. **Use a verified domain** in the `from` address.
7. **Use `upsert` instead of `create`** when syncing users from your application.
8. **Use `bulk`** when sending to multiple recipients at once.
9. **Use imported TypeScript types** (`SendEmailRequest`, `CreateContactRequest`) for type safety.
10. **Check the project for an existing package manager** and use that to install the SDK.

### 6.2 — NEVER DO THE FOLLOWING

1. **Do not** hardcode API keys. Always use environment variables.
2. **Do not** use `from: 'email@domain.com'` — `from` must be `{ email: '...' }` (an object).
3. **Do not** use `to: 'email@domain.com'` — `to` must be `{ email: '...' }` (an object).
4. **Do not** use `html`, `text` alongside `templateId` — they are mutually exclusive.
5. **Do not** use `dynamicData` without `templateId` — it has no effect.
6. **Do not** use `create` when you mean `upsert` — `create` will fail if the contact already exists.
7. **Do not** import from any package other than `autosendjs`.
8. **Do not** use the Resend adapter (`autosendjs/resend`) unless the user is explicitly migrating from Resend.

## Quick Start

Initialize the SDK with your API key:

```typescript
import { Autosend } from 'autosendjs';

const autosend = new Autosend('AS_xxxxxxxxxxxx');
```

<Tip>
	Store your API key in an environment variable (e.g., `AUTOSEND_API_KEY`) rather than hardcoding it
	in your source code.
</Tip>

## Configuration Options

The SDK accepts optional configuration parameters:

```typescript
const autosend = new Autosend('AS_xxxxxxxxxxxx', {
	baseUrl: 'https://api.autosend.com/v1',
	timeout: 30000,
	maxRetries: 3,
	debug: false,
});
```

| Option       | Type    | Default                       | Description                                  |
| ------------ | ------- | ----------------------------- | -------------------------------------------- |
| `baseUrl`    | string  | `https://api.autosend.com/v1` | API base URL                                 |
| `timeout`    | number  | `30000`                       | Request timeout in milliseconds              |
| `maxRetries` | number  | `3`                           | Number of retry attempts for failed requests |
| `debug`      | boolean | `false`                       | Enable debug logging                         |

## Sending Emails

### Plain Text Email

```typescript
await autosend.emails.send({
	from: { email: 'hello@yourdomain.com' },
	to: { email: 'user@example.com' },
	subject: 'Hello World',
	text: 'Welcome to AutoSend!',
});
```

### HTML Email

```typescript
await autosend.emails.send({
	from: { email: 'hello@yourdomain.com', name: 'Your Company' },
	to: { email: 'user@example.com', name: 'John Doe' },
	subject: 'Welcome to Our Platform',
	html: '<h1>Welcome!</h1><p>Thanks for signing up.</p>',
});
```

### Using Templates

Send emails using a pre-built template with dynamic variables:

```typescript
await autosend.emails.send({
	from: { email: 'hello@yourdomain.com' },
	to: { email: 'user@example.com' },
	subject: 'Your Order Confirmation',
	templateId: 'your_template_id',
	dynamicData: {
		name: 'Johnrao',
		orderNumber: '12345',
		orderTotal: '$99.00',
	},
});
```

<Note>
	Learn more about creating and managing templates in the [Email
	Templates](/transactional-emails/email-templates) documentation.
</Note>

## Bulk Emails

Send multiple emails in a single API call:

```typescript
await autosend.emails.bulk({
	from: { email: 'you@example.com' },
	subject: 'Hello',
	html: '<p>Welcome!</p>',
	recipients: [{ email: 'user1@gmail.com' }, { email: 'user2@gmail.com' }],
});
```

<Tip>
	Bulk sending is more efficient than sending emails individually. Use it when you need to send the
	same or similar emails to multiple recipients.
</Tip>

## Sync user data to Contacts on AutoSend

The SDK lets you create, retrieve, update, and delete contacts directly from your code—useful for syncing users from your app, updating [custom fields](/marketing-emails/contacts/custom-fields), or managing [unsubscribe](/others/unsubscribe-groups) preferences.

### Create a Contact

Add a new contact to your AutoSend account. You can assign them to one or more lists and include custom fields for segmentation.

```typescript
await autosend.contacts.create({
	email: 'user@example.com',
	firstName: 'John',
	lastName: 'Doe',
	listIds: ['list_abc123'],
	customFields: {
		company: 'Acme Inc',
		plan: 'pro',
	},
});
```

### Get a Contact

Retrieve a contact's details by their ID. This returns their email, name, list memberships, custom fields, and subscription status.

```typescript
const contact = await autosend.contacts.get('contact_id');
console.log(contact);
```

### Create or Update a Contact

Use `upsert` when you're not sure if a contact already exists. If the email exists, it updates the contact; otherwise, it creates a new one. This is ideal for syncing users from your application.

```typescript
await autosend.contacts.upsert({
	email: 'user@example.com',
	firstName: 'Jane',
	customFields: {
		plan: 'enterprise',
	},
});
```

### Delete a Contact

Permanently remove a contact from your account. This removes them from all lists and deletes their data.

```typescript
await autosend.contacts.delete('contact_id');
```

<Note>
	Learn more about lists, segments, and custom fields in the
	[Contacts](/marketing-emails/contacts/introduction) documentation.
</Note>

## Resend Adapter

If you're migrating from Resend, the SDK provides a compatibility adapter that mirrors the Resend API patterns:

```typescript
import { Resend } from 'autosendjs/resend';

// Use your AutoSend API key
const resend = new Resend('AS_xxxxxxxxxxxx');

// Or use the RESEND_API_KEY environment variable
const resend = new Resend();
```

<Note>
	The Resend adapter uses `properties` instead of `customFields` for contacts, and `remove()`
	instead of `delete()` to match Resend's API conventions.
</Note>

## TypeScript Support

The SDK is written in TypeScript and provides full type definitions out of the box. All methods, parameters, and responses are fully typed for better developer experience and IDE support.

```typescript
import { Autosend, SendEmailRequest } from 'autosendjs';

const autosend = new Autosend('AS_xxxxxxxxxxxx');

const emailRequest: SendEmailRequest = {
	from: { email: 'hello@yourdomain.com' },
	to: { email: 'user@example.com' },
	subject: 'Type-safe email',
	html: '<p>This request is fully typed!</p>',
};

await autosend.emails.send(emailRequest);
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication failed">
    **Possible causes:**
    - Invalid or expired API key
    - API key not included in the request

    **Solutions:**
    - Verify your API key in the <a href={AUTOSEND_PATHS.apiKey} target="_blank" rel="noopener noreferrer">Dashboard</a>
    - Ensure you're passing the API key correctly when initializing the SDK
    - Check that your API key hasn't been revoked

  </Accordion>
  <Accordion title="Request timeout">
    **Possible causes:**
    - Network connectivity issues
    - Request payload too large

    **Solutions:**
    - Increase the timeout in configuration options
    - Check your network connection
    - For bulk emails, reduce the batch size

  </Accordion>
  <Accordion title="Domain not verified">
    **Possible causes:**
    - Sending from an unverified domain
    - DNS records not properly configured

    **Solutions:**
    - Verify your sending domain in the Dashboard
    - Ensure SPF and DKIM records are properly set up
    - Check the [Domain Configuration](/domain) guide

  </Accordion>
  <Accordion title="Rate limit exceeded">
    **Possible causes:**
    - Too many requests in a short period

    **Solutions:**
    - Implement exponential backoff in your application
    - Use bulk sending instead of individual requests
    - Check the [Rate Limits](/api-reference/rate-limit) documentation for your plan's limits

  </Accordion>
</AccordionGroup>
